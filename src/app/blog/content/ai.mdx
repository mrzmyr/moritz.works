## Shared agent instructions

Put shared agent context in `~/.agents`. `system_prompt.md` is the source of truth.

```bash
mkdir -p ~/.agents/skills
touch ~/.agents/system_prompt.md
touch ~/.agents/writing_style.md
```

Different agents read different files:

- Codex reads `AGENTS.md`
- Claude reads `CLAUDE.md`
- Cursor has rules

Symlink them to one shared file.

```bash
ln -s ~/.agents/system_prompt.md ~/.codex/AGENTS.md
ln -s ~/.agents/system_prompt.md ~/.claude/CLAUDE.md
```

Now edits to `~/.agents/system_prompt.md` apply everywhere.

## Writing style

`writing_style.md` is used whenever agents write in your name.

Create `~/.agents/writing_style.md` from existing messages:

- Slack messages
- emails
- PR comments
- docs
- replies

Structure it like this:

```md
# Writing style

## Rules
- English in team channels, German in German 1:1s
- short paragraphs
- direct asks
- use `Context:` and `Impact:` when useful
- avoid "I hope this helps"

## Examples

> can you add @Name to the cursor team?
>
> Impact: Setup security automations for Customer Journey Portal.
```

Then reference it from `~/.agents/system_prompt.md`:

```md
When writing anything in my name, read ~/.agents/writing_style.md first and match the voice, language, and patterns there.
```

## Skills

Skills are small instruction files for specific work.

- [`typescript-principles`](https://github.com/mrzmyr/agent-plugin/blob/main/skills/typescript-principles/SKILL.md): TypeScript defaults for naming, params, and errors.
- [`deslop`](https://github.com/mrzmyr/agent-plugin/blob/main/skills/deslop/SKILL.md): Pushback against fallback-heavy, regex-heavy, custom agent code.

`typescript-principles` covers TypeScript defaults:

- Prefer named parameters over positional parameters
- Prefix boolean functions with `is`, `has`, `can`, `should`, etc.
- Use short action-based function names
- Write structured errors with `status`, `message`, `why`, and `fix`

`deslop` pushes against lazy agent code:

- Question fallbacks
- Question regex parsing when a real parser exists
- Use popular libraries for solved problems
- Avoid type assertions when inference should work

The goal: make taste executable.
