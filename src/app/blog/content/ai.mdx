My AI setup is mostly boring files.

I don't want every agent to become its own isolated context island. Codex has `AGENTS.md`, Claude has `CLAUDE.md`, Cursor has rules, and every tool invents a slightly different place for instructions.

If I copy/paste instructions between them, they drift. So I keep one source of truth and symlink it into each agent.

```bash
ln -s ~/.agents/system_prompt.md ~/.codex/AGENTS.md
ln -s ~/.agents/system_prompt.md ~/.claude/CLAUDE.md
```

Now updates happen once. Every agent gets the same baseline context.

## Writing style

The highest ROI file in my setup is `writing_style.md`.

Agents are decent at writing, but often drift into generic AI voice. Too polished, too long, not how I would actually write.

So I keep a small file with concrete examples from my own messages, comments, docs, and replies. Before an agent drafts anything in my name, it has to read that file.

It includes patterns like:

- English in team channels, German in German 1:1s
- short paragraphs
- direct asks
- use `Context:` and `Impact:` when useful
- avoid "I hope this helps"

It still needs review. But it reduces rewrite time a lot.

## Skills

The second layer is skills.

My baseline instructions should define how the agent behaves. Skills define sharper opinions for specific work.

Two examples:

- [typescript-principles](https://github.com/mrzmyr/agent-plugin/blob/main/skills/typescript-principles/SKILL.md)
- [deslop](https://github.com/mrzmyr/agent-plugin/blob/main/skills/deslop/SKILL.md)

`typescript-principles` encodes small code-style rules that I want repeated everywhere: named parameters over positional parameters, boolean functions prefixed with `is`, `has`, or `can`, short action-based function names, and structured errors that explain `what`, `why`, and `fix`.

`deslop` is a reaction against lazy agent code. Question fallbacks. Question regex parsing when a real parser or platform API exists. Use popular libraries for solved problems. Avoid type assertions when inference should be enough.

The point is not to make the agent memorize my taste. The point is to make taste executable.

## Shape

My setup roughly looks like this:

```txt
~/.agents/
  system_prompt.md
  writing_style.md
  skills/
    typescript-principles/
      SKILL.md
    deslop/
      SKILL.md

~/.codex/AGENTS.md -> ~/.agents/system_prompt.md
~/.claude/CLAUDE.md -> ~/.agents/system_prompt.md
```

The system prompt sets defaults. `writing_style.md` keeps communication close to my actual voice. Skills add local pressure for the work at hand.

Nothing here is fancy. That is the point.

Good agent setup is less about one perfect prompt and more about removing drift.
